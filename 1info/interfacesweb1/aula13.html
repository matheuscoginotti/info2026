Módulo 3: Profundidade

Aula 13: Sombras, Bordas e Elevação

Criando interfaces tridimensionais, cantos suaves e micro-interações dinâmicas com CSS

Lapidando e Elevando as Caixas

Se na aula anterior nós pintamos as caixas e mexemos com transparências, agora vamos tirá-las do papel. No design de interfaces moderno, quinas secas dão lugar a cantos arredondados, e o uso de sombras simula o plano Z (profundidade), fazendo elementos flutuarem e reagirem ao toque humana.

Bordas e Cortes Cirúrgicos

O comando border-radius suaviza as quinas cruas do Box Model. Porém, quando elementos filhos (como imagens) ficam nos cantos, eles ignoram essa curva. É aí que o overflow: hidden; entra, atuando como uma guilhotina que apara o que vazar.

Sombras e Transições

O box-shadow projeta sombras realistas mapeando deslocamento e desfoque. Quando casamos essa elevação com o efeito de :hover controlado por uma transition, criamos a sensação física de que o elemento se mexe.

Regra de Ouro da Estética:
Sombras profissionais são discretas! Evite sombras pretas puras e muito marcadas. Use cores suaves com opacidade baixa (rgba) para criar um efeito realista de iluminação de estúdio.

1. Anatomia da Elevação

Projeção de Sombra

A propriedade box-shadow: X Y blur espalhar cor; adiciona uma ou mais sombras ao redor da caixa, trabalhando perfeitamente a profundidade.

Suavização de Cantos

O comando border-radius: 18px; altera o ângulo reto das quinas. Pode ser aplicado individualmente em cada ponta da caixa.

Contenção de Elementos

Usar overflow: hidden; garante que conteúdos internos não ultrapassem os limites físicos ou as curvas arredondadas da caixa pai.

Animação de Estado

A propriedade transition: 0.3s; dita o tempo que as propriedades de estilo levam para mudar, transformando o atrito em suavidade.

2. Prática: O Card Comercial

Analise abaixo a construção isolada de um card de produto moderno utilizando as técnicas de elevação, arredondamento e filtros de estouro:

<!-- =========================================================
     ARQUIVO: index.html
     ========================================================= -->
<!DOCTYPE html>
<html lang="pt-br">

    <meta charset="UTF-8">
    <title>Card Moderno</title>
    <link rel="stylesheet" href="estilo.css">

<body>

    <div class="card">
        <img src="produto.jpg" alt="Produto">
        <h2>Notebook Gamer</h2>
        <p>Intel Core i7 • 16GB RAM</p>
        <button>Comprar</button>
    </div>

</body>
</html>


/* =========================================================
   ARQUIVO SEPARADO: estilo.css
   ========================================================= */

body {
    background: #f1f5f9;
    font-family: sans-serif;
    padding: 40px;
}

.card {
    max-width: 350px;
    margin: 0 auto;
    background: white;
    border-radius: 18px;
    
    /* GUILHOTINA: Impede que a foto quadrada ignore as curvas do card pai */
    overflow: hidden;
    
    /* SOMBRA: Leve deslocamento vertical (10px) com desfoque amplo (25px) */
    box-shadow: 0 10px 25px rgba(0,0,0,0.20);
    transition: 0.3s;
}

.card:hover {
    transform: translateY(-8px);
    box-shadow: 0 20px 40px rgba(0,0,0,0.30);
}

img {
    width: 100%;
    display: block;
}

h2, p {
    padding: 0 20px;
}

button {
    width: 100%;
    padding: 15px;
    background: #2563eb;
    color: white;
    border: none;
    cursor: pointer;
}

3. Atividade: O Mestre da Profundidade

Execute os seguintes testes práticos modificando o seu arquivo de estilos e responda tecnicamente:

1

Intensidade da Sombra: No seletor da classe .card, altere as configurações do box-shadow para aumentar o desfoque ou a opacidade da cor. Como o elemento reagiu na tela?

2

Isolamento de Curvas: Remova a propriedade global border-radius e teste usar comandos específicos como border-top-left-radius e border-top-right-radius. Conseguimos arredondar apenas o topo?

3

Interação de Botão: Crie a regra para a pseudo-classe button:hover alterando o background para outra cor. O botão mudou instantaneamente ou respeitou alguma regra de transição?

4

Alinhamento Multi-Card: Adicione um segundo bloco <div class="card"> idêntico no HTML. Aplique propriedades de Flexbox no container pai do corpo para posicioná-los lado a lado.

5

Comparações Estruturais: Explique textualmente a diferença prática de comportamento entre a propriedade overflow: hidden; e o próprio border-radius no controle de elementos filhos.
